# Scenario Engine — 시나리오 생성·운영·분석

완성된 시뮬레이션 위에서 워게임을 운영한다. 엔더스파운드리 철학 = **점진적 난이도 상향으로 AI를 단련**한다.

## 시나리오 정의 스키마 (구조화 필수)
시나리오는 재현·공유 가능한 파일이어야 한다. 최소 필드:
```yaml
name: strait_intercept_01
seed: 42
description: 해협 통과 적 함대 요격
blue:                      # 아군 초기 배치
  - {type: destroyer, lat: ..., lon: ..., heading: ..., sensors: [...]}
red:                       # 적군 초기 배치
  - {type: corvette, ...}
objectives:                # 목표
  blue: protect_zone | intercept | escort
  red: transit | strike
roe: weapons_free | weapons_tight    # 교전 규칙
termination:               # 종료 조건
  max_steps: 1000
  win_blue: red_eliminated | zone_held
difficulty: 1              # 난이도 등급 (커리큘럼)
```

## 운영 원칙 (Why)
- **반복 실행 + 통계.** 단판은 운(시드)에 좌우된다. 같은 시나리오를 **N회(예: 30회) 시드 변경 실행**해 승률·생존율·교전효율의 평균·분산을 본다.
- **난이도 곡선 = 커리큘럼.** 쉬운 것부터. AI 승률이 임계(예: 70%)를 넘으면 다음 등급으로: 적 수↑ → 적 정책 고도화 → 환경 악화(기상/야간) → 다대다. 한 번에 어렵게 하면 학습 신호가 죽는다.
- **전술적 해석을 글로.** 숫자만 X. "왜 졌나/이겼나"(예: 조기 탐지 실패, ROE 과보수)를 보고서에 서술 — 이것이 워게임의 진짜 산출물.
- **재현 케이스 보존.** 흥미로운/비정상 대전은 시드+시나리오를 남겨 재현 가능하게.
- **합성 데이터 전제 명시.** 결과는 공개/합성 파라미터 기반이며 실제 교전 결과가 아님을 보고서에 명시.

## 난이도 커리큘럼 설계 (엔더스파운드리 핵심)
| 등급 | 변화 축 | 예 |
|------|---------|-----|
| 1 | 기준 | 1:1, 주간, 단순 적 |
| 2 | 적 수 | 1:2, 적 협동 |
| 3 | 적 정책 | 적이 회피·기만 기동 |
| 4 | 환경 | 야간/악천후, 센서 저하 |
| 5 | 규모 | 다대다 함대전 |
> 승급 조건(직전 등급 승률 임계)과 강등 조건(과적합 방지)을 함께 정의한다.

## 출력
- `scenarios/*.yaml` — 시나리오 정의 파일들
- `_workspace/03_operation_report.md`:
  - 시나리오별 N회 실행 통계 표 (승률·생존율·평균 교전시간)
  - 난이도 곡선 진행 상황
  - 전술 인사이트 (서술형)
  - 재현 케이스 목록 (시드+시나리오)

## 협업
시뮬레이터가 시나리오를 못 처리하면(스키마 불일치 등) `sim-builder`에 결함 보고 후, 처리 가능한 단순 시나리오로 폴백한다.
