# Sim Implementation — 시뮬레이션 구현 컨벤션

아키텍트가 고정한 인터페이스 계약(`01_architecture.md`)을 동작하는 코드로 만든다. **계약을 깨지 않고, 항상 실행 가능한 증분**으로 구축한다.

## 권장 기술 스택 (MVP)
- **언어/런타임:** Python 3.11+
- **환경 API:** `pettingzoo`(멀티에이전트) + `gymnasium`(공간 정의). MVP는 `ParallelEnv`가 단순.
- **수치:** `numpy` (상태·물리 연산). 시드는 `numpy.random.default_rng(seed)`로 캡슐화.
- **상태 저장소:** MVP는 `sqlite3`(mda-gotham `vessels.db`와 호환 스키마). 확장 시 Eclipse Ditto/DTDL 고려.
- **시나리오:** `pyyaml` 또는 JSON.
- **테스트:** `pytest`.
- (이후 확장) RL: `cleanrl`/`rllib` + `pufferlib`; LLM 지휘: `autogen`/`crewai`.

## 디렉토리 구조 (제안)
```

│   ├── agents/      # 규칙기반/RL/LLM 에이전트 (공통 act 인터페이스)

```

## 구현 순서 (MVP 경로 — 매 단계 실행 가능 유지)
1. **상태 모델** — 엔티티 dataclass + SQLite 직렬화. 시드 기반 초기화. → import·왕복(roundtrip) 테스트.
2. **환경 골격** — `reset()`/`step()` 시그니처, 시간 진행 루프(빈 물리). → 스모크 테스트(reset→step N회 무오류).
3. **물리·탐지·교전** — 기동(위치 갱신), 센서 탐지(거리/시야), 교전 판정(확률/규칙). 결정적이어야.
4. **규칙기반 에이전트** — 단순 추격/회피/사격 정책. RL/LLM은 인터페이스만 뚫고 stub.
5. **1판 실행 + 로그** — 시나리오 1개로 끝까지 돌리고 결과 로그/리플레이(JSONL) 출력.
6. **평가 메트릭** — 승패·생존율·교전효율 집계.

## 구현 원칙 (Why)
- **계약 불변.** 아키텍트가 고정한 스키마/시그니처를 임의 변경 금지. 바꿔야 하면 아키텍트에 개정 요청 — 안 그러면 QA가 경계면 불일치로 잡는다.
- **항상 녹색.** 큰 덩어리 한 번에 X. 매 단계 import·실행되는 작은 증분 — 어디서 깨졌는지 즉시 안다.
- **결정성.** 모든 난수는 명시적 시드. 같은 (시드, 시나리오) → 같은 결과. 재현 안 되면 평가·디버깅 불가.
- **순수 함수 지향.** `step()`·`act()`는 부수효과 최소화 — 단위 테스트가 쉬워진다.
- **합성 데이터.** 실제 무기 성능이 아닌 공개/합성 파라미터. 파라미터 출처를 주석으로.

## 검증 연동
각 모듈(상태/환경/에이전트/시나리오) 완성 **직후** `qa-verifier`에 경계면 검증 요청(점진적 QA). 다음 모듈로 넘어가기 전에 불일치를 해소한다.
